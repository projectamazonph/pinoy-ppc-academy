# Assessment and Certification

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

Defines grading, attempts, capstones, certificate issuance, and verification.

## Assessment types

| Type | Purpose | Grading |
| --- | --- | --- |
| Quick check | Confirm one lesson concept | Immediate deterministic answer explanations |
| Knowledge quiz | Test module understanding | Rule-based scoring with question versioning |
| Guided simulation | Teach correct workflow | Step checks, hints, and safe recovery |
| Freeform simulation | Test independent operation | State-based rules and weighted rubric |
| Written diagnosis | Test reasoning and communication | Structured human rubric |
| Capstone | Prove end-to-end readiness | Automated evidence plus instructor review |


## Certification gates

- [ ] All required modules completed.
- [ ] Knowledge score at or above 80 percent.
- [ ] Required simulator scenarios passed.
- [ ] Account audit accepted.
- [ ] Portfolio contains required artifacts and simulated-work disclosure.
- [ ] Capstone rubric passed with no critical safety failure.


## Critical safety failures

- Deleting or archiving the wrong entity in a scenario.
- Using unsafe negative phrase logic that blocks converting queries.
- Ignoring inventory or Buy Box loss while increasing spend.
- Publishing incomplete Sponsored Brands creative.
- Presenting simulated data as real client work.


## Certificate verification

Certificates use a public verification code and show student name, credential, issue date, status, and skill domains. Revoked certificates remain verifiable and display the revoked state without exposing private reasons.
