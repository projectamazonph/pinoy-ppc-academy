# Product Principles

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

The rules used to resolve product and engineering tradeoffs.

## Principles

| Principle | Operational meaning |
| --- | --- |
| Start where the learner is | Do not assume VA, Amazon, advertising, or spreadsheet knowledge |
| Practice before confidence | Confidence should follow completed tasks and feedback |
| Explain the reason | Every scored decision includes why it is safe, risky, or incomplete |
| Evidence over completion | Watching content is not proof of skill |
| Mobile is the default student device | Core lessons, quizzes, and guided simulations must work at 360px width |
| No fake urgency | Pricing and deadlines must be real and administratively recorded |
| No AI dependency | The product must remain useful and complete without LLM services |
| One source of truth | Content, scenarios, rubrics, and pricing must have declared canonical owners |
| Operational simplicity | A solo operator should be able to run the product without a maze of services |


## Decision test

1. Does the change improve a defined learner outcome?
2. Can the result be tested or observed?
3. Does it preserve mobile usability and accessibility?
4. Does it add operational burden for the administrator?
5. Does it create or depend on unverified claims?
6. Can the simpler version produce the same learner outcome?
