# Landing Page Specification

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

Defines the public conversion page, final copy direction, responsive behavior, and test gates.

## Primary message

> **Hero headline:** Start your VA career with a skill clients can actually evaluate.


Supporting copy: Learn Amazon PPC from zero, practice without a live account, and build proof of skill for your first specialized VA role.


Primary action: Start the free VA Career Starter path. Secondary text link: View the full curriculum.


## Page structure

| Section | Purpose |
| --- | --- |
| Sticky header | Brand, curriculum, pricing, sign in, and primary starter CTA |
| Hero | State the audience, transformation, and free first step |
| Audience strip | Show career starters, career shifters, general VAs, and Amazon VAs |
| Pathway | Explain Start -> Learn -> Practice -> Prove -> Apply |
| Why PPC | Explain specialization value without salary guarantees |
| Simulator preview | Show campaign, bid, search term, budget, and audit practice |
| Curriculum | Show free and paid stages with outcomes |
| Proof of skill | Show portfolio, capstone, and verification |
| Founder authority | Use verified work background and product philosophy |
| Pricing | Show one-time tiers and exact inclusions |
| FAQ | Answer beginner concerns, device requirements, time, access, and refunds |
| Final CTA | Return to the free starter path |
| Footer | Policies, contact, product links, and ProjectAmazonPH attribution |


## Visual direction

- Strict ProjectAmazonPH palette: deep navy, sky blue, orange, warm white, and slate neutrals.
- Left-aligned, type-led hero with a real application preview rather than stock photography.
- No decorative gradients, glass cards, neon effects, or generic three-card feature grids.
- Use screenshots from the actual product once available. Before then, use purposeful wireframe previews.
- One dominant action per viewport.


## Trust rules

- [ ] No invented student count, rating, testimonial, salary, or placement statistic.
- [ ] Founder claims must be verified before publication.
- [ ] Pricing and refund terms match the live commerce configuration.
- [ ] Simulators are described as practice environments, not Amazon products.


## Landing page release gates

- [ ] Responsive at 360, 390, 768, 1024, and 1440 pixels.
- [ ] Keyboard navigation and visible focus states pass.
- [ ] No critical WCAG 2.2 AA violations.
- [ ] Lighthouse targets: Performance 90+, Accessibility 95+, Best Practices 95+, SEO 95+ on production-like build.
- [ ] Primary CTA tracking is verified without collecting unnecessary personal data.
- [ ] All claims and links are reviewed manually before launch.
