# Brand and Design System

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

Defines ProjectAmazonPH visual rules for marketing, learning, simulator, and admin surfaces.

## Brand palette

| Token | Value | Use |
| --- | --- | --- |
| Deep navy | #1A1A2E | Primary shell, footer, authority, dark surfaces |
| Sky blue | #007EFF | Learning progress, links, selected controls, informational states |
| Orange | #FF6B35 | Primary marketing CTA, active emphasis, key conversion actions |
| Warm white | #FAFAF7 | Main reading surface |
| White | #FFFFFF | Cards, tables, elevated panels |
| Ink | #171717 | Primary text |
| Slate | #5F6368 | Secondary text and metadata |
| Success | #0E7C3A | Passed, complete, safe |
| Warning | #B45309 | Needs attention, insufficient data |
| Danger | #B91C1C | Destructive, failed, critical risk |


## Typography

- Inter for headings and body in the product and documentation.
- A legible monospace font for metrics, bids, formulas, identifiers, and code.
- Minimum 16px body text on student reading surfaces.
- Compact 14px text is allowed in dense tables when contrast and line height remain accessible.


## Surface rules

- Marketing pages may breathe; application pages are dense and task-oriented.
- Cards use borders before shadows. Avoid nested cards.
- Radii remain restrained: 4px, 6px, and 10px.
- Admin uses tables and inline actions rather than decorative dashboards.
- Motion communicates state, not personality. Respect reduced-motion settings.


## Responsive rules

- Student navigation becomes a compact bottom navigation or drawer on mobile.
- Tables provide column priority, horizontal scroll, or card-row transformation.
- Simulators use guided step mode on phones and analyst workspace mode on larger screens.
- Tap targets are at least 44 by 44 CSS pixels for primary mobile interactions.
