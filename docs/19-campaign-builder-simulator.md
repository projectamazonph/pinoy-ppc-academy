# Campaign Builder Simulator

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

Specifies campaign setup practice across Sponsored Products, Brands, and Display.

## Capabilities

- Build portfolios, campaigns, ad groups, product ads, targets, negatives, budgets, placements, and creatives.
- Validate required fields and relationships before launch.
- Support SP automatic, SP keyword, SP product targeting, SB product collection, SB Store spotlight, SB video, SD contextual, and SD remarketing.
- Preview the final campaign structure and naming before submission.
- Explain validation failures in plain language.


## Scenario examples

| Scenario | Student task |
| --- | --- |
| New product launch | Build research and performance SP campaigns within a fixed budget |
| Branded defense | Separate branded terms and protect branded placement without contaminating generic exact |
| Category expansion | Build product targeting and category tests with clear labels |
| SB video launch | Complete creative, destination, product, and targeting QA |
| SD remarketing | Build views and purchases audiences with correct lookback and product scope |


## Acceptance criteria

- [ ] Invalid relationships cannot be submitted.
- [ ] Required creative fields differ correctly by SB format.
- [ ] SD audience scenarios do not show keyword-only controls.
- [ ] Naming validation reflects the published convention.
- [ ] Every submitted build produces an exportable structure artifact.
