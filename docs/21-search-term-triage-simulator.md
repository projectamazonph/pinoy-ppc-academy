# Search Term Triage Simulator

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

Specifies search-term harvesting, negation, and classification practice.

## Student actions

- Harvest to exact or phrase performance structures.
- Add negative exact or negative phrase.
- Lower a source target bid.
- Leave the term active for more data.
- Flag listing mismatch or product relevance concerns.


## Safety rules

- Negative exact blocks one precise query and is preferred for isolated waste.
- Negative phrase requires stronger evidence because it can block many related queries.
- Harvested terms receive source-campaign negatives only when isolation is part of the scenario strategy.
- Branded queries route to branded defense structures.
- Single-word terms require special caution due to ambiguity and reach.


## Scoring

| Signal | Scoring effect |
| --- | --- |
| Correct intent classification | Positive |
| Correct action with sufficient evidence | Positive |
| Safe negative type | Positive |
| Premature action with weak data | Penalty |
| Blocking a converting query family | Critical penalty |
| Clear written reason | Positive |
