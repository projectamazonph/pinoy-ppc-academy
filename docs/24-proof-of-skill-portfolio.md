# Proof-of-Skill Portfolio

**Product:** Pinoy PPC Academy  
**Repository:** `pinoy-ppc-academy`  
**Version:** 1.0  
**Status:** Documentation baseline

## Purpose

Defines the artifacts students can produce without misrepresenting simulated work.

## Portfolio purpose

The portfolio gives employers evidence of reasoning and execution while protecting clients and avoiding fabricated experience. Every artifact states that it was produced in the Pinoy PPC Academy simulation environment.


## Required artifacts

| Artifact | Evidence shown |
| --- | --- |
| Campaign build | Structure, naming, targeting, budgets, and rationale |
| Search term analysis | Classification, actions, negative safety, and expected impact |
| Bid review | Metric interpretation, decision, calculation, and review window |
| Account audit | Prioritized findings and remediation plan |
| 30-day strategy | Goals, workstreams, budgets, risks, and measurement |
| Client report | What moved, why, actions taken, risks, and next steps |


## Portfolio rules

- [ ] Simulated work label appears on every artifact.
- [ ] No copied client names, ASINs, screenshots, or confidential data.
- [ ] The student explains their own decisions during final review.
- [ ] Artifacts include version and completion date.
- [ ] Only passed artifacts may be published through the platform.


## Export formats

- Public portfolio page with share controls.
- PDF case-study export.
- Structured ZIP containing original CSV, notes, and final report.
- Certificate verification link.
